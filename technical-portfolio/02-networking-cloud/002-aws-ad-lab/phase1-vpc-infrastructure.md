# AWS AD Lab — Infrastructure Setup Guide

> **How to read this guide:**
> Every table below has two columns that matter most:
> - **Field** — the exact label you will see in the AWS Console.
> - **Value / Your Entry** — what to type or select. Where a value is auto-generated by AWS (e.g., an ID), the cell shows a placeholder like `vpc-xxxxxxxxxxxx` with a note to record it after creation.

---

## Phase 1 — Network Infrastructure

### 1.1 Create the VPC

Navigate to: **VPC → Your VPCs → Create VPC**

| Field | Value / Your Entry |
|---|---|
| **Name tag** | `AD-VPC` *(or your own name, e.g. `MyLab-VPC`)* |
| **IPv4 CIDR block** | `10.0.0.0/16` |
| **IPv6 CIDR block** | No IPv6 CIDR block |
| **Tenancy** | Default |
| **VPC ID** *(auto-generated after creation)* | `vpc-xxxxxxxxxxxx` — **record this value** |

> 📝 **Naming tip:** Keep a consistent prefix across all resources in this lab (e.g., `AD-` or your own prefix). It makes filtering in the console much easier.

---

### 1.2 Create Subnets

Navigate to: **VPC → Subnets → Create subnet** *(repeat for each subnet below)*

#### Subnet 1 — Public Subnet (for Ubuntu VPN Server)

| Field | Value / Your Entry |
|---|---|
| **VPC ID** | Select the VPC you just created (`AD-VPC`) |
| **Subnet name** | `AD-Public-Subnet` *(e.g. `MyLab-Public-Subnet`)* |
| **Availability Zone** | `us-east-1a` *(or your preferred AZ)* |
| **IPv4 CIDR block** | `10.0.1.0/24` |
| **Subnet ID** *(auto-generated)* | `subnet-xxxxxxxxxxxx` — **record this value** |
| **Purpose** | Hosts the Ubuntu VPN server — requires internet access |

#### Subnet 2 — Private Subnet (for Windows Domain Controller)

| Field | Value / Your Entry |
|---|---|
| **VPC ID** | Select the same `AD-VPC` |
| **Subnet name** | `AD-Private-Subnet` *(e.g. `MyLab-Private-Subnet`)* |
| **Availability Zone** | `us-east-1a` *(match the public subnet's AZ)* |
| **IPv4 CIDR block** | `10.0.2.0/24` |
| **Subnet ID** *(auto-generated)* | `subnet-xxxxxxxxxxxx` — **record this value** |
| **Purpose** | Hosts the Windows DC — no direct internet; routes outbound through NAT |

---

### 1.3 Create & Attach the Internet Gateway

Navigate to: **VPC → Internet Gateways → Create internet gateway**

| Field | Value / Your Entry |
|---|---|
| **Name tag** | `AD-IGW` *(e.g. `MyLab-IGW`)* |
| **Internet Gateway ID** *(auto-generated)* | `igw-xxxxxxxxxxxx` — **record this value** |

After creation, **attach it to the VPC**:

| Field | Value / Your Entry |
|---|---|
| **Action** | Actions → Attach to VPC |
| **VPC** | Select `AD-VPC` |

---

### 1.4 Create the NAT Gateway *(Optional — enables Windows Updates from private subnet)*

Navigate to: **VPC → NAT Gateways → Create NAT gateway**

**Step 1 — Allocate an Elastic IP first:**
Navigate to **EC2 → Elastic IPs → Allocate Elastic IP address**, then click **Allocate**.

| Field | Value / Your Entry |
|---|---|
| **Elastic IP** | `x.x.x.x` *(auto-assigned)* — **record this value** |

**Step 2 — Create the NAT Gateway:**

| Field | Value / Your Entry |
|---|---|
| **Name** | `AD-NAT-GW` *(e.g. `MyLab-NAT-GW`)* |
| **Subnet** | Select `AD-Public-Subnet` *(must be in the public subnet)* |
| **Connectivity type** | Public |
| **Elastic IP allocation ID** | Select the Elastic IP you just allocated |
| **NAT Gateway ID** *(auto-generated)* | `nat-xxxxxxxxxxxx` — **record this value** |

> ⏳ Wait for the NAT Gateway status to show **Available** before proceeding.

---

### 1.5 Configure Route Tables

Navigate to: **VPC → Route Tables**

#### Route Table 1 — Public-RT (for Public Subnet)

| Field | Value / Your Entry |
|---|---|
| **Name tag** | `AD-Public-RT` *(e.g. `MyLab-Public-RT`)* |
| **VPC** | Select `AD-VPC` |
| **Route Table ID** *(auto-generated)* | `rtb-xxxxxxxxxxxx` — **record this value** |

**Add route:**

| Destination | Target |
|---|---|
| `0.0.0.0/0` | Select `AD-IGW` (your Internet Gateway) |

**Associate with subnet:**

| Field | Value / Your Entry |
|---|---|
| **Subnet associations tab** | Click *Edit subnet associations* |
| **Select** | `AD-Public-Subnet` (`10.0.1.0/24`) |

---

#### Route Table 2 — Private-RT (for Private Subnet)

| Field | Value / Your Entry |
|---|---|
| **Name tag** | `AD-Private-RT` *(e.g. `MyLab-Private-RT`)* |
| **VPC** | Select `AD-VPC` |
| **Route Table ID** *(auto-generated)* | `rtb-xxxxxxxxxxxx` — **record this value** |

**Add route** *(only if NAT Gateway was created)*:

| Destination | Target |
|---|---|
| `0.0.0.0/0` | Select `AD-NAT-GW` (your NAT Gateway) |

**Associate with subnet:**

| Field | Value / Your Entry |
|---|---|
| **Subnet associations tab** | Click *Edit subnet associations* |
| **Select** | `AD-Private-Subnet` (`10.0.2.0/24`) |

## Phase 1.1 — Windows EC2 Instance (Domain Controller)

Navigate to: **EC2 → Instances → Launch instances**

### Step 1 — Name & Tags

| Field | Value / Your Entry |
|---|---|
| **Name** | `W.S25` *(e.g. `MyLab-WinDC`, `WindowsDC-01`)* |

---

### Step 2 — Application and OS Images (AMI)

| Field | Value / Your Entry |
|---|---|
| **Search AMI** | Paste `ami-0fc8a85749a35ce56` in the search bar |
| **OS** | Windows Server 2025 |
| **Architecture** | `64-bit (x86)` |

---

### Step 3 — Instance Type

| Field | Value / Your Entry |
|---|---|
| **Instance type** | `t3.micro` *(1 vCPU, 2 GB RAM)* |

---

### Step 4 — Key Pair

| Field | Value / Your Entry |
|---|---|
| **Key pair name** | `w.s25` *(or your own key name, e.g. `MyLab-WinKey`)* |
| **Key pair type** | RSA |
| **File format** | `.pem` (Linux/macOS) or `.ppk` (Windows/PuTTY) |

> ⚠️ **Keep the .pem / .ppk file safe.** You need it to decrypt the Windows Administrator password.

---

### Step 5 — Network Settings

Click **Edit** in the Network Settings panel.

| Field | Value / Your Entry |
|---|---|
| **VPC** | `AD-VPC` (`vpc-xxxxxxxxxxxx`) |
| **Subnet** | `AD-Public-Subnet` (`subnet-xxxxxxxxxxxx`) — AZ: `us-east-1a` |
| **Auto-assign Public IP** | **Disable** *(instance is in private subnet — no public IP needed)* |
| **Firewall** | Select existing security group |
| **Security group** | `AD-DC-SG` (`sg-xxxxxxxxxxxx`) |

---

### Step 6 — Configure Storage

| Field | Value / Your Entry |
|---|---|
| **Volume type** | `gp3` (recommended) or `gp2` |
| **Device name** | `/dev/sda1` |
| **Size** | `30 GiB` minimum *(increase as needed)* |
| **Delete on termination** | ✅ **Yes** *(check Advanced to confirm)* |

---

### Step 7 — Advanced Details

| Field | Value / Your Entry |
|---|---|
| **IAM instance profile** | `AmazonSSMManagedInstanceCore` |
| **EBS-optimized instance** | **Enabled** |
| **Metadata accessible** | Enabled |
| **Metadata version** | **V2 only (token required)** |
| **Boot mode** | `Legacy BIOS` |
